# DECISIONS.md

중요한 기술적 판단과 이유를 기록합니다. 에이전트는 구현 중 새 결정을 내리면 이 문서에 추가합니다.

## 2026-05-22 - 렌더링 엔진은 Canvas 2D를 기본으로 한다

### 결정

ScriptureFlux의 대규모 교차 참조 선 렌더링은 SVG DOM이 아니라 HTML5 Canvas 2D를 기본으로 구현한다.

### 이유

- 교차 참조 데이터가 수만 개 이상이 될 수 있다.
- SVG path를 수만 개 DOM 노드로 만들면 layout, paint, event handling 비용이 커진다.
- Canvas는 네트워크 배경장을 한 번에 그리는 방식에 적합하다.

### 영향

- 개별 링크 접근성은 별도 UI 또는 hit-test 레이어로 보완해야 한다.
- React 컴포넌트는 렌더링 대상이 아니라 상태와 UI chrome에 집중한다.

## 2026-05-22 - D3는 보조 계산용으로만 사용한다

### 결정

D3는 scale, interpolation, quadtree 같은 계산 보조 도구로만 사용하고, D3 selection 기반 DOM 렌더링은 사용하지 않는다.

### 이유

- React와 D3 DOM 제어가 충돌할 수 있다.
- Canvas 중심 구조에서 D3의 강점은 좌표/스케일/공간 인덱싱에 있다.

## 2026-05-22 - 성경 본문과 교차 참조 데이터는 분리한다

### 결정

교차 참조 관계 데이터와 실제 성경 본문 텍스트 데이터는 별도 파일과 별도 라이선스 문서로 관리한다.

### 이유

- 교차 참조 데이터와 성경 번역 본문은 라이선스 조건이 다를 수 있다.
- 본문 없는 데모도 가능하게 만들어 저작권 위험을 낮춘다.

## 2026-05-22 - 초기 릴리즈는 정적 SPA로 제한한다

### 결정

초기 버전에는 서버, 계정, 로그인, DB, 외부 유료 API를 추가하지 않는다.

### 이유

- 사용자가 요구한 페이지는 정보 교환이 아니라 보여주기에 집중한 정적 웹페이지다.
- GitHub Pages 배포와 오픈소스 공개에 적합하다.

## 2026-05-22 - Canvas 기반 시각화에서의 키보드/터치 접근성 보완

### 결정

캔버스를 포함하는 컨테이너에 `tabIndex={0}`과 포커스 스타일을 부여하고, 키보드 방향키(`ArrowLeft/Right/Up/Down`) 및 모바일 터치 드래그(`onTouchStart`, `onTouchMove`) 이벤트를 통해 활성 교차 참조(active link)를 조작할 수 있도록 설계한다.

### 이유

- HTML5 Canvas는 그래픽 렌더링에 특화되어 있으나 내부 개별 선이 DOM 요소가 아니므로 브라우저의 기본 포커스 및 터치 이벤트를 받지 못한다.
- 웹 접근성(a11y) 가이드라인을 준수하기 위해 컨테이너 포커스를 통한 키보드 운용성을 확보하고, 시각적 포커스 링 및 힌트 UI를 제공하여 사용자 경험을 premium하게 만든다.

## 2026-05-22 - GitHub Actions와 GitHub Pages를 통한 지속적 빌드 및 배포

### 결정

GitHub Actions 기반 CI/CD 파이프라인(`.github/workflows/deploy.yml`)을 구축하여, `main` 브랜치 푸시 시 자동으로 Lint, Typecheck, Test, Build를 진행하고 성공 시 GitHub Pages로 배포하도록 연동한다.

### 이유

- 수동 배포로 인한 환경 차이와 배포 오류를 차단하고 빌드 성공 품질을 완벽히 담보하기 위함이다.
- 정적 SPA 아키텍처에 가장 이상적이며 오픈소스 생태계에 유기적으로 기여하는 구조이다.

## 2026-05-22 - 120ms 호버 디바운스 및 클릭 기반 핀(Pin) 고정 메커니즘을 통한 인터랙션 안정성 확보

### 결정

마우스 호버 시 세부 정보 카드 뷰가 파르르 요동치는 현상을 방지하기 위해 120ms 디바운스 딜레이(Dwell)를 적용하고, 캔버스 내 참조선 클릭 시 활성 참조선을 골드 네온 라인으로 영구 고정(Pin)하는 `pinnedLink` 상태를 도입한다.

### 이유

- 마우스를 성경 축 또는 곡선 영역에서 연속적으로 이동할 때 실시간 카드 갱신에 의해 눈의 피로와 시각적 깜빡임 현상이 발생한다. 120ms의 디바운싱을 통해 미세한 포인터 잔물결을 안정화한다.
- 사용자가 원하는 특정 교차 참조 정보를 고정한 채로 텍스트를 읽고 싶을 때, 포인터 고정이 가능하도록 클릭 이벤트를 추가하여 핀(Pin) 기능을 제공하고 핀 해제 전까지 시각적 glow 및 텍스트 카드를 고정 유지한다.

## 2026-05-22 - 퍼블릭 도메인 한국어 성경(개역한글 1961) 탑재 및 한/영 다국어 듀얼 레이아웃 개편

### 결정

한국인 사용자 경험 극대화를 위해 대한민국 저작재산권 보호기간(50년)이 만료된 퍼블릭 도메인 성경인 **개역한글(1961년판)** 번역본을 로컬 JSON 데이터셋으로 탑재하고, `ReferenceCard` 상단에 한국어를 크게 배치하며 하단에 영문 KJV 번역본을 점선 구분 및 이탤릭 서체로 병기하는 한/영 다국어 듀얼 레이아웃을 개편한다.

### 이유

- 국문 성경 본문 렌더링 요청을 충족하되, public 저장소의 법적/라이선스 위험성을 완전히 제거하기 위해 완전한 퍼블릭 도메인 판본인 '개역한글(1961)'을 선택했다.
- 영어(en) KJV를 보조 자료로 결합 제공함으로써 신뢰도 높고 학술 가치가 높은 듀얼 레이아웃을 구성하여 프리미엄 UX를 완수한다.

## 2026-05-22 - 모바일 60FPS 최적화 (DPR 1.5 제한 클램핑 및 18px 터치 히트 반경 튜닝)

### 결정

모바일 기기의 high-DPI 렌더링 부하를 줄이기 위해 Device Pixel Ratio(DPR)를 최대 1.5로 Clamping 제한하고, 핑거 터치 환경의 조작 편의를 위해 터치 히트 반경(Hit Test Radius)을 기존 8px에서 18px로 확장한다.

### 이유

- 3x 이상의 초고해상도 모바일 디스플레이에서 Canvas 2D 연산량이 비약적으로 상승하여 프레임 드랍이 발생한다. 눈으로 구분하기 힘든 DPR 1.5 초과 영역을 제한함으로써 모바일 60FPS 성능을 견고히 보장한다.
- 마우스와 달리 굵은 손가락 터치는 좁은 오프셋 좌표로 곡선을 정밀 탭하기 어려우므로 모바일 터치 히트 반경을 너그럽게 조정한다.

## 2026-05-22 - 시각적 예술 영감 감사 헌사 (Chris Harrison & OpenBible.info)의 UI/문서 통합

### 결정

본 프로젝트의 시각 예술적 근원인 Chris Harrison 교수와 교차 참조 오픈 데이터를 제공한 OpenBible.info의 Sean Harrison 및 Christoph Römhild 목사에게 경의를 표하기 위해 UI 상단 패널, 하단 푸터, README.md, ATTRIBUTION.md에 감사 헌사 문구 및 하이퍼링크를 상세히 수록한다.

### 이유

- ScriptureFlux는 독자적인 상용 서비스가 아니라 시각화 예술의 아름다움을 정적 웹앱으로 구현 및 공유하고자 하는 목적을 갖는다. 오리지널 아이디어와 데이터 가치를 존중하는 것은 오픈소스 및 학술적 도덕성의 기초이자 요구 조건이다.

## 2026-05-22 - clientWidth/Height 기반 캔버스 크기 정밀 매핑 및 책별 분할 JSON 2단계 LOD 레이지 로딩 아키텍처 도입

### 결정

- 캔버스가 부모 크기를 벗어나 바닥이 잘리고 위에만 나오던 결함을 `ResizeObserver` 상에서 `entry.target.clientWidth/Height` 기반으로 논리 픽셀을 정밀하게 추출하고, `<canvas>`의 인라인 CSS 스타일을 `width: '100%', height: '100%'`로 강제 바인딩하여 해결한다.
- 11,907쌍의 대규모 교차 참조와 66권 전체 장/절 성경 구절 데이터를 브라우저 렉 없이 초고속 로드하기 위해 책별 동적 청크 분할(`public/data/bible-text/` 및 `public/data/cross-references/`) 전처리 파이프라인(`scripts/prepare-data.js`)을 구축한다.
- 최초 로딩 시에는 가벼운 랜드마크 교차 참조선 파일 (`public/data/cross-references.json`, 16쌍)만 탑재(LOD Level 1)하고, 성경 책 영역 호버/액티브 시 해당 책의 세부 참조 JSON 데이터를 비동기 fetch하여 머지(LOD Level 2)하는 2단계 LOD 캔버스 렌더러를 구축한다.
- `ReferenceCard`에서도 성경 구절 본문을 통째로 임포트하던 메모리 낭비 구조를 철폐하고, 구절 정보 활성화 시 책 인덱스에 매칭되는 책 본문 JSON 파일만 dynamic fetch하고 메모리 캐싱 및 미려한 로딩 스켈레톤 UI를 보여주도록 전면 개편한다.

### 이유

- 캔버스의 실제 물리 크기와 화면상의 논리 크기 매핑 시 픽셀 단위 누락이나 DPR 곱연산 불일치로 잘림이 발생하는 결함을 근본적으로 예방하고, 모든 스크린 배율에서 항상 컨테이너에 꽉 찬 레이아웃을 보장하기 위함이다.
- 수십 메가바이트에 이르는 전체 데이터셋을 싱글 번들로 패키징할 경우, 웹앱의 초기 로딩 성능이 최악으로 치닫고 모바일 기기 메모리가 고갈된다. 데이터셋을 66권 단위로 잘게 파쇄하여 필요한 시점에만 동적으로 load/merge 함으로써 극도의 가벼움과 60FPS의 쾌적한 렌더링 성능을 영구히 유지한다.

### 영향

- 초기 로드되는 데이터 자산 용량이 99% 이상 감소하여 모바일 환경의 네트워크 트래픽과 로드 타임이 경이적으로 줄어들었다.
- 120ms 호버 디바운싱 필터링과 비동기 fetch 병렬 호출, 로컬 메모리 캐시 조합으로 끊김 없는 유기적 인터랙션 감각을 완수했다.
